Occtet Curator – User Guide

1. Introduction

1.1 What is Occtet Curator?

Occtet Curator is a professional platform designed to analyze, curate, and assess SBOMs (Software Bill of Materials), as well as to generate VEX reports. An SBOM provides a detailed inventory of all software components within a project. This information is essential for: - Security analysis (CVEs and vulnerabilities) - License compliance - Copyright analysis - Risk assessment within the software supply chain Occtet Curator transforms a raw SBOM into structured, reviewable units that can be assessed by humans. The core entity in this process is the InventoryItem.

1.2 Key Concepts

Term Description
SBOM A list of all software components used in a project (e.g., SPDX).
InventoryItem A structured representation of a software component, including associated files, licenses, copyrights, and CVEs.
SoftwareComponent The actual software component (e.g., OpenSSL 3.0.2).
Vulnerability / CVE A standardized identifier indicating whether a software component is affected by a known vulnerability.
VEX A statement describing whether a vulnerability (CVE) is relevant to the project.
AuditNote Documentation used to justify and record review decisions.

2. Workflow: How to Work with Occtet

Occtet Curator follows a project-based structure. An organization can manage multiple users and projects. The administrator is responsible for: - Creating organizations - Assigning users to organizations - Defining the system path where source code is stored A curator typically works as follows: 1. Create a project (Projects) 2. Import an SBOM (Import → SPDX Import, or via ORT Starter if an ORT instance is available) 3. Open SBOM Management (SBOMmgmt), select the project, and begin curation: - InventoryItems are displayed on the right - The file tree is displayed on the left 4. Use the Overview page to review vulnerabilities, licenses, and copyrights 5. Work with InventoryItems: - Each item contains detailed information about the software component - Includes vulnerabilities, licenses, and copyrights - Contains associated files or standalone files grouped by shared attributes (e.g., same license) - AuditNotes can be added to document evidence and decisions 6. Create and download VEX reports for vulnerabilities directly within the context of an InventoryItem 7. Download the curated SBOM from the project page The system is designed to convert an unverified SBOM into a validated, audit-ready software inventory. The following sections describe each part of the system in more detail.

3. Configuration (Admin)

3.1 System Path Setup

The system path defines where project source code is stored. This is required to enable file content viewing within SBOM Management. Without a system path, projects cannot be created.

3.2 Text Autocompletion (Suggestions)

By default, a set of predefined suggestions for text autocompletion is available. These can be used in AuditNotes or as names for InventoryItems. Only administrators can create new suggestions. Currently, two specific contexts are supported. These should be used as intended, as they are explicitly implemented for these use cases.

3.3 Search Term Profiles

Search term profiles can be defined to highlight specific keywords within file contents. Common examples include: itext, gpl, lgpl, stackoverflow. Multiple search terms should be entered as a comma-separated list.

4. Organization (Admin)

Administrators create organizations and assign users to them. Projects can also be assigned to organizations. - When an administrator creates a project, they can select the target organization. - When a curator creates a project, it is automatically assigned to their organization.

5. Projects

5.1 Creating a Project

The following fields are required when creating a project: - Name - Version - Product Owner Optional fields: - Contact email - Search profile Project assignment rules: - Admin-created projects: organization can be selected - Curator-created projects: organization is assigned automatically Data within an organization can be reused across its projects. Licenses and copyrights are not restricted to a specific organization and can be reused globally.
