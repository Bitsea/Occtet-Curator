Occtet Curator – Guía de usuario

1. Introducción

1.1 ¿Qué es Occtet Curator?

Occtet Curator es una plataforma profesional diseñada para analizar, curar y evaluar SBOMs (Software Bill of Materials), así como para generar informes VEX. Una SBOM describe todos los componentes de software que forman parte de un proyecto. Esta información es fundamental para: - Análisis de seguridad (CVE y vulnerabilidades) - Cumplimiento de licencias - Análisis de derechos de autor - Evaluación de riesgos en la cadena de suministro de software Occtet Curator transforma una SBOM sin procesar en unidades estructuradas y verificables que pueden ser revisadas por personas. El elemento central en este proceso es el InventoryItem.

1.2 Conceptos clave

Término Descripción
SBOM Lista de todos los componentes de software de un proyecto (por ejemplo, SPDX).
InventoryItem Representación estructurada de un componente de software, incluyendo archivos, licencias, derechos de autor y CVEs.
SoftwareComponent El componente de software en sí (por ejemplo, OpenSSL 3.0.2).
Vulnerability / CVE Identificador estándar que indica si un componente de software está afectado por una vulnerabilidad conocida.
VEX Indica si una vulnerabilidad (CVE) es relevante para el proyecto.
AuditNote Documentación utilizada para justificar y registrar decisiones.

2. Flujo de trabajo: cómo trabajar con Occtet

Occtet Curator sigue un enfoque basado en proyectos. Una organización puede contener varios usuarios y proyectos. El administrador es responsable de: - Crear organizaciones - Asignar usuarios a las organizaciones - Definir la ruta del sistema donde se almacena el código fuente El curador trabaja normalmente de la siguiente manera: 1. Crear un proyecto (Projects) 2. Importar una SBOM (Import → SPDX Import o mediante ORT Starter si hay una instancia ORT disponible) 3. Acceder a SBOM Management (SBOMmgmt), seleccionar el proyecto y comenzar la curación: - InventoryItems a la derecha - Árbol de archivos a la izquierda 4. Revisar en la vista Overview las vulnerabilidades, licencias y derechos de autor 5. Trabajar con los InventoryItems: - Contienen información detallada sobre los SoftwareComponents - Incluyen vulnerabilidades, licencias y derechos de autor - Agrupan archivos asociados o archivos independientes con características comunes (por ejemplo, la misma licencia) - Permiten añadir AuditNotes para documentar decisiones y evidencias 6. Crear y descargar informes VEX en el contexto de un InventoryItem 7. Descargar la SBOM curada desde la página del proyecto El sistema está diseñado para convertir una SBOM sin procesar en un inventario de software verificado y listo para auditoría. Las siguientes secciones describen las distintas áreas del sistema con más detalle.

3. Configuración (Admin)

3.1 Definir la ruta del sistema

La ruta del sistema determina dónde se almacena el código fuente de los proyectos. Es necesaria para poder visualizar el contenido de los archivos en SBOM Management. Sin esta ruta, no se pueden crear proyectos.

3.2 Autocompletado de texto (sugerencias)

Por defecto, existen algunas sugerencias de autocompletado. Estas pueden utilizarse en AuditNotes o como nombre de InventoryItems. Solo el administrador puede crear nuevas sugerencias. Actualmente existen dos contextos específicos que deben utilizarse tal como están definidos.

3.3 Términos de búsqueda

Se pueden definir perfiles de términos de búsqueda para resaltar palabras clave en el contenido de los archivos. Ejemplos comunes: itext, gpl, lgpl, stackoverflow. Los términos deben introducirse como una lista separada por comas.

4. Organización (Admin)

El administrador crea organizaciones y asigna usuarios a ellas. También puede asignar proyectos a organizaciones. - Si el administrador crea un proyecto, puede seleccionar la organización - Si un curador crea un proyecto, se asigna automáticamente a su organización

5. Proyectos

5.1 Crear un proyecto

Los siguientes campos son obligatorios: - Nombre - Versión - Responsable del producto Campos opcionales: - Correo electrónico de contacto - Perfil de búsqueda Los datos de los proyectos dentro de una organización pueden reutilizarse. Las licencias y los derechos de autor no están ligados a una organización específica y pueden utilizarse globalmente.

6. Importar una SBOM

Puedes importar una SBOM (actualmente en formato SPDX) mediante la función de carga de archivos. Durante la importación, también puedes configurar el uso de flujos de trabajo basados en IA para: - detección de derechos de autor - clasificación de licencias Tras la importación, Occtet genera automáticamente: - SoftwareComponents - InventoryItems - Vulnerabilidades (CVEs) - Archivos - Licencias - Derechos de autor

7. Iniciar un ORT Run

En la página de importación también encontrarás un botón para iniciar un ORT run. Para utilizar esta función, es necesario que haya una instancia de ORT (Docker) en ejecución e integrada en el sistema. Los datos de conexión pueden configurarse en el archivo application.properties o mediante un archivo .env. Al configurar un ORT run, debes definir: - el tipo de repositorio a analizar - la URL del repositorio - la versión

8. Panel de control

El panel de control actúa como un indicador de seguridad del proyecto. Muestra, entre otros: - el número de CVEs por nivel de gravedad (Critical, High, Medium, Low) - cuántos componentes de software presentan vulnerabilidades Esto te permite evaluar rápidamente si un proyecto está listo para su lanzamiento o si representa un riesgo.

9. Gestión de SBOM

Este es tu espacio de trabajo principal: - izquierda: árbol de archivos - derecha: InventoryItems - centro: Overview (vulnerabilidades, licencias, derechos de autor) Idealmente, cada archivo debe asignarse a un único InventoryItem.

10. InventoryItems – Elemento central

Un InventoryItem representa un componente de software e incluye: - SoftwareComponent (nombre, versión, URL de origen) - archivos asociados - vulnerabilidades (CVEs) (con posibilidad de crear entradas VEX) - licencias - derechos de autor - AuditNotes para documentar decisiones Los InventoryItems son la base de tu análisis y permiten dividir la SBOM en unidades manejables.

11. Trabajo con archivos

11.1 Visor de código

Puedes: - abrir archivos - leerlos - buscarlos - asignarlos a un InventoryItem Durante la revisión, puedes encontrar información relevante como: - encabezados de licencia - avisos de copyright - referencias a código de terceros Esta información debe documentarse en las AuditNotes para garantizar la trazabilidad de las decisiones. También deben registrarse posibles restricciones de exportación u otras notas legales detectadas. Los términos de búsqueda definidos en tu perfil se resaltan en el contenido.

11.2 Estado “Reviewed”

Un archivo se considera revisado cuando: - está asignado correctamente - su licencia ha sido verificada - sus derechos de autor han sido revisados - toda la información ha sido documentada Una vez completado, se marca como reviewed.

12. AuditNotes

Las AuditNotes sirven como documentación y evidencia. Permiten registrar, por ejemplo: - por qué una licencia es aceptable - por qué una CVE no es relevante - por qué un copyright puede ignorarse Son esenciales para: - auditorías (por ejemplo, ISO) - transparencia ante clientes - documentación legal

13. Vulnerabilidades y VEX

Occtet detecta automáticamente las CVEs asociadas a cada SoftwareComponent. Puedes evaluarlas mediante estados VEX: - Affected - Not Affected - Under Investigation - Fixed Estas evaluaciones pueden exportarse en formato VEX (Vulnerability Exploitability Exchange), legible por máquinas.

14. Vistas y resúmenes

Occtet ofrece varias vistas globales:
Vista Propósito
Inventory Todos los InventoryItems
VEX Listing Todas las decisiones VEX
Licenses Todas las licencias del proyecto
SoftwareComponents Componentes técnicos
Projects Todos los proyectos
Todas las listas pueden filtrarse, ordenarse y buscarse.

15. Exportación

Puedes exportar: - SBOMs depuradas (incluyendo información legal relevante como CVEs, licencias y derechos de autor) - documentos VEX Esto permite: - entregar resultados a clientes - cumplir con requisitos de auditoría o autoridades

16. Integración

Gracias a su API y a la integración con ORT, Occtet Curator puede incorporarse en pipelines de construcción. Esto permite ejecutar ORT runs automáticamente e importar los resultados directamente. También se detectan ejecuciones iniciadas externamente, que pueden importarse fácilmente. De este modo, puedes mantener tus SBOM actualizadas y evaluar continuamente las vulnerabilidades.
